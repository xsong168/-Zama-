# 强制规则：存储避雷

## 文件命名铁律

### ✅ 强制执行
**所有文件名必须使用：英文 + 时间戳格式**

### 命名规范

#### 1. 标准格式
```
YYYYMMDD-描述-版本.扩展名
```

#### 2. 示例
- ✅ `20250215-fix-filename-encoding-error.md`
- ✅ `20250215-audio-tuning-methodology.md`
- ✅ `20250215-automotive-industry-script-v1.mp3`
- ❌ `修复文件名编码报错.md`（禁止中文）
- ❌ `音频调教.md`（禁止中文）

#### 3. 目录命名
- **根目录**: 必须英文（如 `Junshi_Bot`）
- **子目录**: 可使用中文（如 `本体画像/`、`记忆库/`）
- **原因**: Git 和终端工具对根目录路径敏感

### 避雷原因

#### Windows 编码问题
- PowerShell 对中文路径处理不稳定
- Git 操作可能出现 `PathNotFound` 警告
- 跨平台协作会遇到字符集冲突

#### 跨平台兼容性
- Linux/macOS 默认 UTF-8
- Windows 默认 GBK/CP936
- 文件名中文会导致哈希值不一致

#### 自动化脚本友好
- CI/CD 管道通常不支持中文路径
- 自动化备份脚本可能失败
- 日志解析工具无法识别

### 特殊场景例外

#### 可使用中文的场景
- 内容正文（Markdown 文件内部）
- 临时测试文件（不纳入版本控制）
- 最终交付的视频/音频标题（面向用户）

#### 必须使用英文的场景
- Git 仓库中的所有文件名
- 系统配置文件（`.env`, `bot.py`）
- 自动化脚本路径
- 数据库备份文件

### 执行检查清单

提交前必须验证：
- [ ] 文件名是否包含中文？
- [ ] 是否有时间戳前缀？
- [ ] 扩展名是否正确？
- [ ] Git status 是否正常显示？

### 违规处理
**一旦发现中文文件名，立即重命名并重新提交。**
