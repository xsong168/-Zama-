# 情景记忆：修复文件名编码报错

**时间**: 2025-02-15  
**问题**: Windows PowerShell 无法正确处理中文路径，导致 Git 操作报错  
**严重程度**: 中等（影响 Git 版本控制）

## 问题现象
```
Set-Location : 找不到路径"C:\Users\GIGABYTE\Desktop\Junshi_Bot冷酷军师"
```

## 根本原因
- PowerShell 默认编码与中文路径存在兼容性问题
- Windows 系统对 Unicode 路径的处理不一致

## 解决方案
虽然 Git 操作最终能够执行（工作目录正确），但输出日志存在编码警告。

**最佳实践**：
1. ✅ 项目根目录避免使用中文命名
2. ✅ 内部子目录可使用中文（如 `本体画像/`、`记忆库/`）
3. ✅ 关键文件名强制使用英文+时间戳格式

## 实施结果
- ✅ Git 仓库成功初始化
- ✅ 文件正常追踪
- ✅ 提交历史干净（commit `78f68aa`）

## 教训提炼
**物理避雷原则**：关键路径和文件名必须使用英文，避免跨平台编码问题。
